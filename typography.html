Adobe® Source Sans Pro

Source Sans Pro is a well thought out open source typeface developed with user interfaces in mind. We like it for its clean, modern letterforms, and the wide range of weights it offers. It also has a smidgen of character—adding some friendliness without venturing too far from neutral.

The Weights

Source Sans Pro Extralight, Source Sans Pro Light, Source Sans Pro Regular, Source Sans Pro Semibold, Source Sans Pro Bold, Source Sans Pro Black...Pretty sweet, huh?

Download Font Files

ZIP file includes all 6 font weights in WOFF, EOT, TTF, and SVG formats. A CSS file is also included for convenience.

A Few Tech Details

Source Sans Pro is smaller when compared (pixel to pixel) to other webfonts. Text and headline sizes have been adjusted (up) in the CSS to visually appear equal in height to Arial (Fig. 1). Time was taken to figure out sizes from 9px to 100px.

Other Useful Information

  • Source Sans Pro is used for headlines and body copy.
  • The CSS references WOFF, EOT, TTF formats directly from Google with local versions as backup. SVG format (if needed) is still served locally.
  • All font sizes are in REMs (root ems) with their sizing relative to the base 16px/100% browser default. Pixel assignments are provided for older browsers.

How It Looks

On websites, content is king and should be treated as such. Without good typographic practices, readability and comprehension could suffer—deposing the once kingly content to the level of visual noise. So, we have carefully crafted type sizing, weights, and spacing to help keep content on the throne. Long live content!

About Headlines:

  • <h1> to <h6> tags are not given any default sizes in the CSS. Sizing is assigned using “headXX” classes where the “XX” is the desired pixel height.
  • Default font weight is Source Sans Pro Regular. Weights can be overridden by adding “ultra” “heavy” “semibold” “regular” “lite” “extralite” classes.
  • Size classes range from 12px to 100px. View full list in “Size Chart” tab.

What Lists Look Like

Have you ever had a few short points to make, but didn’t want bury them in a massive paragraph? Then try an ordered or unordered list!

  • Unordered Lists
  • Ordered Lists
  • Feature Lists

A Few Rules About Lists:

  • A list needs to have at least two bullet points/sub-bullets. If you only have a single bullet point, find another way to display your content.
  • Be consistent. Write all your bullet points as a sentence or a fragment. Pick one and stick with it.
text16ul
  • Sell products—This is so vague as to be meaningless and is likely to produce buzzword-infested fluff.
  • Sell this product—Selling a product is a process made up of many smaller tasks, like discussing benefits, mapping them to features, demonstrating results and value, and asking people to buy. If your goal is this vague, you have no idea which of these tasks (if any) the content will perform.
  • List and demonstrate the benefits of this product—This is something a chunk of content can actually do. But if you don’t know who is supposed to benefit from the product, it’s difficult to be specific.
    • Who is the product made for?
    • How does it help or simplify their lives?
    • Why would they want to buy from you?
  • Show how this product helps nurse practitioners—If you can discover what nurse practitioners need, you can create content that serves this purpose. (And if you can’t find out what they need before trying to sell them a product, you have a lot more to worry about than your content.)

  • Programmatic access through RESTful APIs
